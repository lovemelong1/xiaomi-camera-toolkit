# 常见问题

## 为什么默认不合并完整日录像？

完整日录像需要读取一天内所有视频片段，并重新编码成一个新视频。NAS 的 CPU 和硬盘 I/O 通常比较弱，一天录像可能要跑很久。

延时摄影只抽少量帧，速度快很多，也更适合长期归档。

## 10 秒延时摄影能看到一整天吗？

可以。工具会覆盖 00:00 到 24:00，但抽帧权重不同：

- 00:00-06:00 权重低
- 06:00-18:00 权重高
- 18:00-24:00 权重中等

这样夜里不会占太多帧，白天变化会更清楚。

## 删除源文件安全吗？

开启 `DELETE_SOURCE=true` 后，工具只会在对应日期输出成功后删除源文件。

如果设置 `KEEP_DAYS=15`，最近 15 天不会删除，只删除更早的源文件。

## 能处理多个摄像头吗？

可以。只要录像文件在 `INPUT_DIR` 下面，并且文件名/目录名符合支持格式，工具会自动扫描。

如果两个摄像头同一天都有录像，目前会合并到同一个日期输出。需要按摄像头分别输出时，建议给每个摄像头运行一个独立容器，分别挂载不同输入/输出目录。

## 输出视频在哪里？

延时摄影：

```text
OUTPUT_DIR/延时摄影/YYYYMM/YYYYMMDD.mp4
```

每日完整录像：

```text
OUTPUT_DIR/日期录像/YYYYMM/YYYYMMDD.mp4
```

## 没有生成某一天怎么办？

常见原因：

- 那一天没有匹配到支持格式的视频文件
- 源视频损坏，ffmpeg 无法读取
- 输出目录没有写入权限
- Docker 挂载路径写错

可以先用 `DRY_RUN=true` 查看扫描和输出计划。

## 如何只保留延时摄影功能？

设置：

```text
MODE=timelapse
```

这是默认值，也是推荐值。
