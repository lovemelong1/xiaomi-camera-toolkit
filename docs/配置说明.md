# 配置说明

所有配置都可以通过命令行参数或环境变量传入。Docker 使用时建议用环境变量。

## 基础路径

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `INPUT_DIR` | `/data/input` | 源录像目录 |
| `OUTPUT_DIR` | `/data/output` | 输出目录 |

## 运行模式

| 变量 | 默认值 | 可选值 | 说明 |
| --- | --- | --- | --- |
| `MODE` | `timelapse` | `timelapse`、`merge-day`、`both` | 处理模式 |
| `RUN_MODE` | `once` | `once`、`scheduler` | 跑一次或常驻定时 |
| `SCHEDULE_TIME` | `03:00` | `HH:MM` | 常驻模式运行时间 |
| `INTERVAL_DAYS` | `1` | 正整数 | 常驻模式间隔天数 |
| `TARGET_DAY` | 空 | `YYYYMMDD` | 手动指定处理哪一天 |
| `TARGET_LAG_DAYS` | `1` | 正整数 | 常驻模式处理几天前 |

模式说明：

- `timelapse`：只生成延时摄影，推荐。
- `merge-day`：只生成每日完整压缩录像，速度较慢。
- `both`：同时生成延时摄影和每日完整压缩录像。

常驻模式默认只处理前一天。例如 2026-07-21 03:00 运行时，目标日期是 2026-07-20。如果这一天没有源文件，会记录跳过，不会继续往前生成 2026-07-19。

## 延时摄影参数

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `FRAMES` | `30` | 每天抽帧数量 |
| `FPS` | `3` | 输出帧率 |
| `WIDTH` | `1280` | 输出视频宽度 |

输出时长大约等于：

```text
FRAMES / FPS
```

例如：

```text
30 / 3 = 10 秒
```

## 删除参数

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `DELETE_SOURCE` | `false` | 是否删除源文件 |
| `KEEP_DAYS` | 空 | 只删除超过 N 天的源文件 |
| `DELETE_EMPTY_DIRS` | `true` | 删除源文件后清理空目录 |

删除逻辑是保守的：

1. 先按天生成输出。
2. 确认这一天输出有效。
3. 如果开启 `DELETE_SOURCE=true`，再检查 `KEEP_DAYS`。
4. 只有超过保留天数的源文件才会删除。

## 覆盖和演练

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `OVERWRITE` | `false` | 已有有效输出时是否重新生成 |
| `DRY_RUN` | `false` | 只打印计划，不真正生成或删除 |

首次使用建议先跑：

```bash
docker run --rm \
  -v /volume3/16TB/xiaomi_camera_videos:/data/input \
  -v /volume3/16TB/合并小米录像:/data/output \
  -e DRY_RUN=true \
  xiaomi-camera-toolkit
```

确认扫描到的天数和输出路径没问题，再关闭 `DRY_RUN`。
